Media Queries

Media Queries

With media queries we can ask for certain types of output and additional features. Depending on those we can change styles for these features.
Media Queries are stylesheets within stylesheets.

Only the rules within the media query are changing. Everything else stays the same.

Features

Look at the stylesheet for this document. Use the develper tools to inspect what's happening.

You can test lots of things with attributes:

  • width
    the viewport width
  • height
    the viewport height
  • orientation
    landscape or portrait orientation of the device
  • aspect-ratio
    the ration of width to height

The are many more conditions to test. Read on Media Queries.

Using device pixels

Pixels are relative! Compare the resolution of your desktop and your phone. CSS-pixels are rendered on your device. If you develop on your desktop and make your viewport smaller or larger, it's still on your desktop! It won't work on a handheld device.

Use your meta-tag to set the CSS-width equal to the device-width.

Syntax

@media type (features){
    /*rules for thes features*/
}
If you don't include a type it will target to all types (screen, print, ...)
For the features, mostly you will ref a certain width.
min-width mean from this width and larger, so it's for a larg(er) screen. max-width works the other way around. You can also work within a range between min-width and max-width.

Working with min-width means you are working desktop-first. The order of media queries is important! If you work with min-width, you work from the smallest to the biggest.

Try the whole stylesheet mobile-first!

Organizing media queries within a stylesheet

There is no right or wrong way to organize media queries. Be consistent! You can write all rules within one media query or you can write a media query just after each separate rule that needs one.

Breakpoints

Breakpoints are the width at which we want to change the design.
Don't use standard breakpoints. Better, look at the screen and decide at what point does the layout look wrong. It depends on your design. Try different breakpoints. Don't target specific devices, because there are just too many.

For maintainability keep the breakpoints to a minimum. The more breakpoints you have, the more adjustments you have to make, if you are making changes to on breakpoint.
